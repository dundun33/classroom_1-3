# 1-3 Classroomまとめ — 週次更新ルール（Claude Code用）

このリポジトリは、1年3組（82回生）向けの保護者用ダッシュボードです。
Google Classroom／ロイロノートの投稿と週の時間割を、LINEで共有できる1枚のHTMLにまとめます。

## ファイル構成（2ファイル方式）

- `index.html` … デザイン本体（**原則さわらない**。`window.CLASSROOM_DATA`を読む）
- `data.js` … **毎回更新するのはこれだけ**（`updateISO` / `updateTime` / `timetable` / `items`）
- `inbox/` … 更新の入力を置く場所（Classroomのコピーテキスト、ロイロのスクショ、時間割PDF）

index.htmlのデザインを変える指示が明示された時だけindex.htmlを編集する。それ以外は **data.jsのみ** を書き換える。

## 毎回の作業手順（Claude Code）

1. `inbox/` の中身を読む（テキスト・画像・PDF）。
2. 入力の先頭にあるコピー時刻（例「6/16 15:28にコピー」）か、スクショのiPhoneステータスバーの時刻を読み、それを基準日時にする。**時刻が分からなければ止めて聞く**。
3. 下のルールに従って `data.js` の `updateISO` / `updateTime` / `timetable` / `items` を作り直す。
4. 検証：`node -e "global.window={};require('./data.js');console.log(window.CLASSROOM_DATA.items.length)"` がエラーなく動くこと。
5. `git add data.js && git commit -m "update: <基準日> 反映" && git push`（GitHub Pagesが自動反映）。
6. 反映した内容と、判断に迷った点を1〜2行で要約して報告する。

## data.js のスキーマ

```js
window.CLASSROOM_DATA = {
  updateISO: "2026-06-16",   // 基準日（New判定・相対日付・グレーアウトの起点）
  updateTime: "15:28",       // 基準時刻（曜日は自動付与）
  timetable: {
    label: "6/15(月)〜6/19(金)",
    days: [ {n:'月',md:'6/15',iso:'2026-06-15'}, ... ],   // 月〜金（土は授業がある週だけ）
    rows: [  // 時限1〜6 × 曜日。セル = {s,t,r,mk?,ref?,rs?} または null
      [ {s:'数',t:'森本',r:'HR'}, {s:'読書会',mk:'持ち物',ref:'n4',rs:2}, ... ],
      [ ..., null, ... ],   // 上のrsで覆われるマスはnull
    ]
  },
  items: [
    { id:'h1', cat:'hw',         // cat: 'hw'=宿題・提出物 / 'no'=連絡事項
      date:'2026-06-16',         // 締切日(YYYY-MM-DD)。無い項目は省略
      dateLabel:'6/16 (火) 16:10',// 表示用ラベル
      optional:true,             // 任意参加なら付ける（希望者タグ・希望者を隠す対象）
      subject:'数学',            // 教科。'部活'は「部活を隠す」対象
      title:'…', details:['…','…'],
      thread:'1年 数学',         // スレッド名（下のリンク表のキー）。ロイロは 'ロイロ'
      poster:'森本奈央', posted:'6/15' },  // ロイロは poster:'' / posted無し
  ]
};
```

## 内容の振り分け

- 2セクション構成：「1. 宿題・提出物」(`cat:'hw'`) と「2. 連絡事項」(`cat:'no'`)。各セクション内は締切順。
- 任意参加の催し・応募は**連絡事項のみ**に入れ `optional:true`（宿題・提出物には入れない）。
- 生徒が出す提出物は、当番が回収する形でも宿題・提出物（`cat:'hw'`）に入れる（チェックで消せるように）。
- 締切のないお知らせ（結果配布・確認事項など）も**基本入れる**。保護者が把握でき、完了チェックで隠せるため。
- 他クラス宛て（例「美術4組」）など3組に無関係なものは除外。3・6組のように3組を含むものは入れる。
- Classroomの提出物とロイロの同一課題は1行に統合（締切が明確な方を採用、スレッド欄に両方明記）。

## 過去項目の保持（重要）

- **未来**の項目は残す。**直近1週間以内（基準日の約7日前まで）の過去**も残す（グレー表示。「過去を隠す」で消せる）。
- 1週間より古い過去だけ削除する。
- 作り直すたびに、未来・直近過去・常設項目（部活/常時/当面の間/次回の授業/夏休み）を**取りこぼさず引き継ぐ**。
  （過去に未来の部活予定や直近過去の連絡を誤って落としたミスがあったので注意）

## 日付ラベルと並び順

- ラベル例：「6/16 (火) 16:10」「6/19 (金) まで」「次回の授業」「常時」「当面の間」「夏休み」「今週中」。
- 日付ありは昇順、その後に日付なし。日付なしの順序：次回の授業 → 当面の間 → 夏休み → 常時。

## Newバッジ

- 投稿日が基準日(`updateISO`)から約24時間以内、**かつ過去項目でない**時だけ付く（グレーアウト項目には付けない）。
- ロイロ行は poster/posted 無し＝Newは付かない。

## ロイロノート

- 赤い「募集中」だけを宿題・提出物(`cat:'hw'`)の行にする。「提出済」「締切超過」は捨てる。
- 教科＝青く選択中の教科名（先頭の番号は除く）。スレッドは 'ロイロ'（リンクなし、プレーン表示）。poster/posted は空。

## 時間割

- 学校の平常時間割表PDFから、**3組（各曜日の3つ目の組サブ列）だけ**を抽出。
- 表は 時限1〜6 × 曜日（月〜金。土は授業がある週だけ）。教科（太字）＋ 教室/先生（小）。
- 教室名はPDF表記のまま（HR, HR/数メ, 理科A/B, 声楽器楽, 被服B, 二体/一体, 美術室, 声楽, LL/400 など）。
- 当日が表示週内なら、その曜日の列が自動でアンバー強調される。

### 連続ブロックの結合（rowspan）
- 火1・2限「読書会」のように、PDFで1つのセルとして縦に連続するブロックは、上のセルに `rs:N` を付け、覆われる下のマスを `null` にする。
- 同じ教科が偶然連続するだけ（例：水5・6限の美、木5・6限の理）は**結合しない**（PDFでも別セル）。`rs`は本当に1ブロックの時だけ。

### 時間割↔連絡のリンク
- 投稿が「特定の授業中の出来事」（例「6/11(木)の授業で小テスト」、持ち物、授業内容変更）を含む時、その曜日・時限のセルに
  `mk`（短いバッジ文言。アンバーのピル）と `ref`（対応するitemのid）を付ける。
- index.html側で、`ref`付きセルはタップでそのセクションを開き、`#row-<id>` へスクロールして行を一瞬光らせる。

## やってはいけない / 確認

- index.htmlの不要な改変、デザインの勝手な変更はしない。
- 判断に本当に迷う項目（締切不明・対象クラス不明など）は、削る前に要約で確認する。
- 認証情報・パスワードの入力や、権限変更・恒久ルール作成はしない。
