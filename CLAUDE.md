# 1-3 Classroomまとめ — 週次更新ルール（Claude Code用）

このリポジトリは、1年3組（82回生）向けの保護者用ダッシュボードです。
Google Classroom／ロイロノートの投稿と週の時間割を、LINEで共有できる1枚のHTMLにまとめます。

## ファイル構成（2ファイル方式）

- `index.html` … デザイン本体（**原則さわらない**。`window.CLASSROOM_DATA`を読む）
- `data.js` … **毎回更新するのはこれだけ**（`updateISO` / `updateTime` / `timetable` / `items`）
- `inbox/` … 更新の入力を置く場所（Classroomのコピーテキスト、ロイロのスクショ、時間割PDF）

index.htmlのデザインを変える指示が明示された時だけindex.htmlを編集する。それ以外は **data.jsのみ** を書き換える。

## 毎回の作業手順（Claude Code）

1. `inbox/` の中身を**すべて**読む（テキスト・画像・PDF）。**新旧ファイルが混在していてもOK。全部読んでから判断する**（古いファイルがあっても止めない・無視しない）。
   - 基準日時は**最も新しい材料**から取る（最新のコピー時刻／スクショの時刻）。
   - 時間割PDFが複数あれば、**対象週が最新（＝これから来る週）のもの**を採用。ファイル名の日付（例`0707.pdf`）や表内の日付で判断。
   - **時間割PDFやロイロのスクショがinboxに無いのは正常（毎回入れるとは限らない）。無い材料は「既存のdata.jsの該当ブロックをそのまま維持」するだけで、絶対に止めない・失敗扱いにしない。** classroom_posts.txt が1つあれば items の更新は必ず実行できる。
   - 同じ投稿が複数材料に重複する時は**新しい方を採用**し、内容をマージ。
2. 入力の先頭にあるコピー時刻（例「6/16 15:28にコピー」「取得時刻 2026/07/04 19:45」）か、スクショのiPhoneステータスバーの時刻を読み、それを基準日時にする。
   - 本文に時刻の記載が無いファイル（時刻ヘッダーの無い生コピペ等）は、**ファイルのmtime（更新日時、`ls -la`で確認）を代わりに使う**。複数材料がある場合は、本文記載の時刻とmtimeを横並びで比較し、最も新しいものを基準日時にする。
   - **古いファイルがinboxに残っていること自体を理由に止めない**（古い材料は補助情報として使うだけで、基準日時の判定はできる）。本文にもmtimeにも手がかりが無く、他の材料との前後関係も判断できない場合に限り、止めて聞く。
3. 下のルールに従って `data.js` の `updateISO` / `updateTime` / `timetable` / `items` を作り直す。
4. 検証：`node -e "global.window={};require('./data.js');console.log(window.CLASSROOM_DATA.items.length)"` がエラーなく動くこと。
5. `git add data.js && git commit -m "update: <基準日> 反映" && git push`（GitHub Pagesが自動反映）。
6. 反映した内容と、判断に迷った点を1〜2行で要約して報告する。

## data.js のスキーマ

```js
window.CLASSROOM_DATA = {
  updateISO: "2026-06-16",   // 基準日（New判定・相対日付・グレーアウトの起点）
  updateTime: "15:28",       // 基準時刻（曜日は自動付与）
  timetable: {
    label: "6/15(月)〜6/19(金)",
    days: [ {n:'月',md:'6/15',iso:'2026-06-15'}, ... ],   // 月〜金（土は授業がある週だけ）
    rows: [  // 時限1〜6 × 曜日。セル = {s,t,r,mk?,ref?,rs?} または null
      [ {s:'数',t:'森本',r:'HR'}, {s:'読書会',mk:'持ち物',ref:'n4',rs:2}, ... ],
      [ ..., null, ... ],   // 上のrsで覆われるマスはnull
    ]
  },
  items: [
    { id:'h1', cat:'hw',         // cat: 'hw'=宿題・提出物 / 'no'=連絡事項
      date:'2026-06-16',         // 締切日(YYYY-MM-DD)。無い項目は省略
      dateLabel:'6/16 (火) 16:10',// 表示用ラベル
      optional:true,             // 任意参加なら付ける（希望者タグ・希望者を隠す対象）
      subject:'数学',            // 教科。'部活'は「部活を隠す」対象
      title:'…', details:['…','…'],
      thread:'数学',             // スレッド名（下のリンク表のキー）。**投稿元スレッド**（classroom_posts.txtでその投稿が入っていたコース見出し）を必ず使う。教科(subject)につられて書き換えない。数学は必ず'数学'表記('1年 数学'は使わない）。ロイロは 'ロイロ'
      poster:'森本奈央', posted:'6/15',   // ロイロは poster:'' / posted無し
      url:'https://classroom.google.com/c/.../p/... または /m/.../details' },  // 全アイテムに付ける。/m/(要確認)ならピルも表示、スレッド名は常にこのURLへ直リンク
  ]
};
```

## 内容の振り分け

- 2セクション構成：「1. 宿題・提出物」(`cat:'hw'`) と「2. 連絡事項」(`cat:'no'`)。各セクション内は締切順。
- 任意参加の催し・応募は**連絡事項のみ**に入れ `optional:true`（宿題・提出物には入れない）。
- 生徒が出す提出物は、当番が回収する形でも宿題・提出物（`cat:'hw'`）に入れる（チェックで消せるように）。
- 締切のないお知らせ（結果配布・確認事項など）も**基本入れる**。保護者が把握でき、完了チェックで隠せるため。
- 他クラス宛て（例「美術4組」）など3組に無関係なものは除外。3・6組のように3組を含むものは入れる。
- Classroomの提出物とロイロの同一課題は1行に統合（締切が明確な方を採用、スレッド欄に両方明記）。

## 要確認アイテム（資料/課題タイプ）の扱い

- `classroom_posts.txt` の「資料/課題（要確認）」セクションの各行は、本文が取得できていない（詳細ページの中にある）投稿。
- これらは内容がタイトルから明確に推測できる場合（例:「◯◯テスト範囲」）でも本文を創作せず、itemの`title`にそのタイトルを使い、`details`は空か「詳細はClassroomの投稿を確認」の一言にする。
- **各アイテムに url を必ず付ける**：`classroom_posts.txt` でそのアイテムの元になった投稿の「URL:」をそのまま採用する。平文投稿は `/p/…`、要確認（資料/課題）は `/m/…/details`。同じ投稿者の似た投稿が複数ある時は、投稿日（posted）と本文で正しい1件を選ぶ。
- index.html の挙動：スレッド列のスレッド名リンクは、url があればその投稿へ直リンク（無ければスレッドのトップ）。タイトル横の「詳細 ↗」ピルは、url が `/m/`（一段深い詳細ページを持つ要確認）の時だけ表示する。平文（`/p/`）はピルを出さない。
- 振り分けは内容から推測：テスト範囲・カレンダー等の情報系は連絡事項(`cat:'no'`)、提出を伴いそうなものは宿題(`cat:'hw'`)。迷ったら連絡事項に入れて希望者タグは付けない。
- **投稿者・日付**：要確認行にある `投稿者: ◯◯` と `日付: 作成 ◯◯` を、item の `poster` / `posted` に入れる。`posted` は「作成 」を外し、「昨日」「◯日前」等の相対表記は**取得時刻（先頭の取得時刻行）を基準に `M/D` へ変換**する（例: 取得時刻が7/5なら「作成 昨日」→ `posted:'7/4'`）。投稿者が `(取得できず)` の要確認行は `poster` を省略してよい。
- Classroom由来でない項目（ロイロ、バスケ部の予定、手動で足した項目など）は元URLが無いので url は付けない（スレッド名は従来どおりそのスレッドのトップに飛ぶ）。
- **thread は必ず「投稿元スレッド」**（`classroom_posts.txt`でその投稿が入っていた`========== コース: ◯◯ ==========`見出し）を使う。教科(`subject`)と投稿元が違う場合（例：数学の内容が82回生スレッドに投稿されている）でも、`thread`は投稿元のまま変えない。`subject`につられて書き換えるミスをしないこと。`url`のコースID（`/c/`の後ろ）と`thread`は必ず一致させる。

## 過去項目の保持（重要）

- **未来**の項目は残す。**直近1週間以内（基準日の約7日前まで）の過去**も残す（グレー表示。「過去を隠す」で消せる）。
- 1週間より古い過去だけ削除する。
- 作り直すたびに、未来・直近過去・常設項目（部活/常時/当面の間/次回の授業/夏休み）を**取りこぼさず引き継ぐ**。
  （過去に未来の部活予定や直近過去の連絡を誤って落としたミスがあったので注意）

## 日付ラベルと並び順

- ラベル例：「6/16 (火) 16:10」「6/19 (金) まで」「次回の授業」「常時」「当面の間」「夏休み」「今週中」。
- 日付ありは昇順、その後に日付なし。日付なしの順序：次回の授業 → 当面の間 → 夏休み → 常時。

## Newバッジ

- 投稿日が基準日(`updateISO`)から約24時間以内、**かつ過去項目でない**時だけ付く（グレーアウト項目には付けない）。
- ロイロ行は poster/posted 無し＝Newは付かない。

## ロイロノート

- 赤い「募集中」だけを宿題・提出物(`cat:'hw'`)の行にする。「提出済」「締切超過」は捨てる。
- 教科＝青く選択中の教科名（先頭の番号は除く）。スレッドは 'ロイロ'（リンクなし、プレーン表示）。poster/posted は空。

## 時間割

- 学校の平常時間割表PDFから、**3組（各曜日の3つ目の組サブ列）だけ**を抽出。
- 表は 時限1〜6 × 曜日（月〜金。土は授業がある週だけ）。教科（太字）＋ 教室/先生（小）。
- 教室名はPDF表記のまま（HR, HR/数メ, 理科A/B, 声楽器楽, 被服B, 二体/一体, 美術室, 声楽, LL/400 など）。
- 当日が表示週内なら、その曜日の列が自動でアンバー強調される。

### ⚠️ PDFが複数ページに割れている場合（重要・過去に誤読）
- このPDFは**本来1枚の表**で、作成者がExcel→PDF時に気を利かせず溢れさせただけ。2ページ目は
  1ページ目と**同じ「曜日×6組」の列構成のまま**、時限5・6の行が下に続いているだけ。
- **必ず両ページを1つの表として縦に連結して読む**（余白は無視）。1ページ目＝時限1〜4、2ページ目＝時限5〜6。
- ページごとに列を数え直すと、**結合セル（MHC交流会・学年学活・ICT・自治の会など曜日/組をまたぐ大セル）で列がズレる**。
  「何番目の単語か」で数えず、**列のx座標（3組の帯）で固定して縦に読む**のが安全。
- pdfplumberでの確実な抽出例（3組の帯だけ縦読み。座標帯は表により微調整）：
  `python3 -c "import pdfplumber;p=pdfplumber.open('inbox/xxxx.pdf');[print(round(w['top'],1),w['text']) for pg in p.pages for w in pg.extract_words() if 559.5<=(w['x0']+w['x1'])/2<=591.5]"`
- 組み上げたら**必ず保護者（ユーザー）に表を見せて検算**してから push（行事系セルは特に要確認）。

### 連続ブロックの結合（rowspan）
- 火1・2限「読書会」のように、PDFで1つのセルとして縦に連続するブロックは、上のセルに `rs:N` を付け、覆われる下のマスを `null` にする。
- 同じ教科が偶然連続するだけ（例：水5・6限の美、木5・6限の理）は**結合しない**（PDFでも別セル）。`rs`は本当に1ブロックの時だけ。

### 時間割↔連絡のリンク
- 投稿が「特定の授業中の出来事」（例「6/11(木)の授業で小テスト」、持ち物、授業内容変更）を含む時、その曜日・時限のセルに
  `mk`（短いバッジ文言。アンバーのピル）と `ref`（対応するitemのid）を付ける。
- index.html側で、`ref`付きセルはタップでそのセクションを開き、`#row-<id>` へスクロールして行を一瞬光らせる。

## やってはいけない / 確認

- index.htmlの不要な改変、デザインの勝手な変更はしない。
- 判断に本当に迷う項目（締切不明・対象クラス不明など）は、削る前に要約で確認する。
- 認証情報・パスワードの入力や、権限変更・恒久ルール作成はしない。
